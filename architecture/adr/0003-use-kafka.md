# 3. Использовать Kafka для асинхронных интеграций

Дата: 2026-06-30

## Статус

Принято

## Контекст

Сценарий оформления заказа включает несколько независимых этапов: бронирование ингредиентов, оплату, приготовление и обновление статуса заказа. Не все этапы должны выполняться синхронно в одном запросе пользователя.

## Решение

Использовать Kafka для публикации бизнес-событий между сервисами.

## Последствия

Положительные последствия:

- снижение связанности сервисов;
- возможность асинхронной обработки;
- повышение устойчивости при временной недоступности потребителей.

Отрицательные последствия:

- требуется идемпотентность consumer'ов;
- требуется мониторинг очередей и ошибок обработки;
- модель данных становится eventually consistent.
