# 2. Использовать API Gateway

Дата: 2026-06-30

## Статус

Принято

## Контекст

К системе обращаются iOS App, Android App, Web App и Web Terminal. Клиентским приложениям не нужно знать внутреннюю топологию микросервисов.

## Решение

Использовать API Gateway как единую точку входа во внутреннюю микросервисную архитектуру.

## Последствия

Положительные последствия:

- единая точка маршрутизации;
- централизованная проверка авторизации;
- скрытие внутренней структуры системы от клиентов.

Отрицательные последствия:

- API Gateway становится критичным компонентом;
- требуется контролировать его отказоустойчивость и масштабирование.
