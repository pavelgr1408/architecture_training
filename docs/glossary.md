# Глоссарий

| Термин | Значение |
|---|---|
| C4 Model | Подход к описанию архитектуры через уровни Context, Container, Component и Code. |
| Container | Исполняемая часть системы: приложение, сервис, база данных, очередь или другое хранилище. |
| Component | Внутренняя часть контейнера, показывающая устройство сервиса. |
| ADR | Architecture Decision Record — запись архитектурного решения. |
| API Gateway | Единая точка входа для клиентских приложений. |
| Eventual consistency | Модель согласованности, при которой данные становятся согласованными не мгновенно, а после обработки событий. |
| Kafka | Платформа потоковой передачи событий, используемая для асинхронных интеграций. |
