## Цель проекта

Цель проекта — показать архитектуру платформы оформления заказов еды с использованием микросервисного подхода.

В проекте описаны:

- контекст системы;
- контейнерная архитектура;
- компонентная архитектура Order Service;
- синхронные и асинхронные интеграции;
- архитектурные решения;
- нефункциональные требования.

## Как читать документацию

1. Начните с раздела **System Context**.
2. Затем перейдите к **Container Diagram**.
3. После этого изучите **Component Diagram — Order Service**.
4. Для обоснования решений посмотрите раздел **Архитектурные решения**.
