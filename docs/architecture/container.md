# Container Diagram

## Назначение диаграммы

Container Diagram показывает внутреннее устройство Food Ordering Platform на уровне приложений, микросервисов, баз данных и внешних интеграций.

## Контейнеры

| Контейнер | Ответственность |
|---|---|
| iOS App | Клиентское мобильное приложение для iOS. |
| Android App | Клиентское мобильное приложение для Android. |
| Web App | Клиентское веб-приложение. |
| Web Terminal | Веб-интерфейс для оператора. |
| API Gateway | Единая точка входа и маршрутизация запросов. |
| Order Service | Корзина, оформление заказа и статусы заказа. |
| Menu Service | Меню и доступность блюд. |
| Ingredient Service | Ингредиенты, остатки и бронирование. |
| Auth Service | Авторизация и проверка токенов. |
| Payment Service | Оплата заказа. |
| Robot Service | Передача заказов на роботизированное приготовление. |

## Архитектурный принцип

Каждый микросервис владеет своей предметной областью и собственным хранилищем данных. Межсервисные взаимодействия делятся на синхронные REST-запросы и асинхронные события Kafka.
