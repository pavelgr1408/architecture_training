# ADR-0003. Использовать Kafka для асинхронных интеграций

## Статус

Принято.

## Контекст

Сценарий оформления заказа включает оплату и приготовление. Эти шаги могут занимать время и не должны блокировать пользовательский HTTP-запрос до полного завершения процесса.

## Решение

Использовать Kafka для асинхронных бизнес-событий между сервисами.

Ключевые события:

- OrderPaymentRequested;
- PaymentCompleted;
- PaymentFailed;
- OrderCookingRequested;
- CookingCompleted;
- CookingFailed.

## Последствия

Положительные последствия:

- сервисы становятся менее связанными;
- временная недоступность consumer'а не ломает producer;
- процесс заказа можно развивать поэтапно.

Отрицательные последствия:

- нужна идемпотентная обработка событий;
- нужна стратегия повторных попыток;
- данные становятся eventually consistent.
