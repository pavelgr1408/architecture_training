# ADR-0002. Использовать API Gateway

## Статус

Принято.

## Контекст

К системе обращаются разные типы клиентов: iOS App, Android App, Web App и Web Terminal. Клиентские приложения не должны знать внутренние адреса и структуру микросервисов.

## Решение

Использовать API Gateway как единую точку входа.

API Gateway отвечает за:

- маршрутизацию запросов;
- первичную проверку авторизации;
- скрытие внутренней топологии микросервисов от клиентов.

## Последствия

Положительные последствия:

- упрощается клиентская интеграция;
- внутренние сервисы не публикуются напрямую наружу;
- появляется единое место для cross-cutting concerns.

Отрицательные последствия:

- API Gateway становится критичным компонентом;
- требуется масштабирование и мониторинг API Gateway.
